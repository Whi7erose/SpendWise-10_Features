How It Works
1

Map Opens from Alert or Dashboard

Launched via a pre-purchase alert tap (Feature 01) or from the dashboard. Current GPS position is centred; 600m radius is scanned for participating retailers.

2

Retailer Discount Data Fetched

SpendWise queries the B2B retailer API for all stores within radius — returning store name, active promotions, and estimated basket price for the user's top purchased categories.

3

Price Gap Calculated & Colour-Coded

Historical spend at the user's usual store is compared to current prices at each nearby store. LKR saving per trip is calculated and pins are coloured: green = save significantly, gold = slight saving, red = your current.

4

One-Tap Navigation Handoff

Tapping a pin opens the detail card showing the active discount, estimated saving, and distance. "Get Directions" hands off to Google Maps with the destination pre-loaded.

Traceability

This functionality is implemented as a map-based extension of UC3 and serves the Geolocation-Based Saver persona. It connects directly to the Porter Value Chain feedback loop: retailers provide discount data → SpendWise notifies users → users choose lower‑cost nearby stores → retailers gain increased footfall → the cycle reinforces itself.

⚠ Prototype Limitation

The map uses a live OpenStreetMap embed styled for the dark theme. The pins and interactions are CSS-rendered over the map to demonstrate the intended UI flow. Final version uses the Google Maps JavaScript SDK with dynamic overlay markers. Retailer pricing data is seeded — live version requires a B2B retailer API updated daily by participating merchants.
