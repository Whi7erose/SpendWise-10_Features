How the Parsing Engine Works
1

SMS Listener On-Device

Android SMS READ_SMS permission triggers on every incoming message. iOS uses email parsing (bank e-notifications). Raw SMS text is captured on-device before any processing begins.

2

Bank Identification

The sender ID ("SAMPATH", "COMBANK", "SEYLAN-BANK") is matched against a bank pattern library. Each bank has its own SMS template registered the parser knows exactly which regex to apply.

3

NLP Field Extraction

Named Entity Recognition extracts: amount (regex + currency normalisation), merchant name (cleaned of store codes), date/time (normalised to ISO), and transaction type (debit/credit).

4

Category Classification

Merchant name is matched against SpendWise's 12,000+ Sri Lankan merchant→category lookup. Unrecognised merchants go to an ML classifier trained on merchant name patterns.

5

Feed Entry + Budget Check

Transaction posted to live feed. Budget comparison runs instantly if a category goes over limit, a budget alert fires. If near a retail location, the pre-purchase alert pipeline (Feature 01) activates.

Traceability

Implements FR-01 (automatic parsing of bank SMS messages) and satisfies the NFR‑Performance requirement (≤ 3‑second parse time). This capability forms the core transaction ingestion flow, from SMS receipt through to dashboard entry. By eliminating manual input, it directly tackles the high dropout rate observed in budgeting applications, addressing the primary reason users abandon such tools.

⚠ Prototype Limitation

The "Simulate SMS" button adds pre-built sample transactions to demonstrate the feed UI. In production, real transactions arrive via Android SMS_RECEIVED broadcast receiver. The NLP engine and merchant category lookup require a trained model and database not included in this prototype.
