How It Works Step by Step
1

GPS Geofence Trigger

User walks within 150m of a registered retail location (Keells, Cargills, Arpico, etc.). Background location service fires a geofence entry event using Android/iOS significant-location-change API minimal battery impact.

2

Budget Status Check

App instantly checks if the user's Groceries category is within 20% of their monthly budget limit. If budget is healthy, no alert fires zero notification fatigue. If approaching the limit, the alert pipeline activates.

3

Competitor Price API Query

SpendWise queries its retail discount database updated daily via B2B partnerships with participating retailers. Pulls current promotions within 600m for the user's most-purchased product categories.

4

Push Notification Fired

If a cheaper alternative exists, a rich push notification is sent showing the saving in LKR, store name, and distance. Tapping opens the in-app Savings Map (Feature 04) with walking directions pre-loaded.

5

User Decision + Feedback Loop

User taps "Go to Cargills" or "Stay at Keells". Either choice is logged. If a user ignores 5+ alerts for the same store pair, alert frequency is intelligently reduced suppression prevents annoyance over time.

⚠ Prototype Limitation

The notification is simulated as a static UI card. Full production requires: Firebase Cloud Messaging for push delivery, Background Geofence SDK (Google Awareness API / iOS Core Location), and a live retailer discount database with a daily update pipeline.
