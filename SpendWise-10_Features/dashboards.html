How AI Classification Works
1

Transaction Ingestion — Days 1–30

All auto-parsed SMS transactions are stored with merchant, category, time-of-day, day-of-week, and location metadata. The first 30 days build the baseline profile — no persona is assigned yet; user sees a generic dashboard.

2

Behaviour Vector Scoring

Algorithm scores the user on 5 axes: location-sensitivity, budget-discipline, category-concentration, grocery-frequency, and discount-redemption rate. The highest-scoring axis determines the primary persona assignment.

3

Dashboard Layout Personalisation

Persona determines which widgets and charts appear first. Geolocation Savers see savings trends and store comparisons; Financial Allocators see budget usage bars and spend-vs-target lines; Grocery Optimisers see store price comparisons front and centre.

4

Weekly Re-Classification

Persona is re-evaluated every 7 days. If spending behaviour shifts — e.g. user starts using more discounts — persona updates automatically. Users can also manually override their persona assignment in Settings.

Traceability

The five personas are derived from the defined target market. The personalisation engine implements FR-02 (personalised insights), aligns with CSF-03 (algorithm quality driving user retention), and demonstrates the SWOT SO-1 strategy by leveraging AI to create switching costs against competitors.

⚠ Prototype Limitation

Clicking a persona shows a pre-designed dashboard state with simulated chart data. Full production requires a trained ML model (k-means clustering or decision tree on transaction vectors), real transaction data, and a backend inference API that drives chart data per user session in real time.
